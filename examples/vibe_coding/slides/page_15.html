AI 활용 패턴 Interaction Patterns

AI를 단순한 도구가 아닌 동료 개발자처럼 대하는 4가지 방법

1
명확한 요구사항 전달
"이메일, 비밀번호, 이름을 받아서 User 모델로 만들고, 비번은 bcrypt로 해싱해서 MongoDB에 저장해줘"
2
컨텍스트 제공
"기존 프로젝트 구조의 get_mongodb_client() 함수를 참고해서 UserRepository를 구현해줘"
3
점진적 개선
1. "일단 User 모델만 만들어줘" → 2. "이제 Repository 추가해줘" → 3. "서비스 로직 구현해줘"
4
함께 개선하기
"이 코드 괜찮은데, 비밀번호 검증 로직은 별도 private method로 분리하는 게 좋을 것 같아"

Before & After

나쁜 질문 (Vague)

"유저 생성 기능 만들어줘"

→ AI가 마음대로 구조를 잡고, 프로젝트 스타일과 맞지 않는 코드를 생성할 확률이 높습니다.

좋은 질문 (Specific)

"User 도메인 모델을 기반으로 이메일 중복 체크가 포함된 서비스 로직을 짜줘"

→ 구체적인 요구사항과 컨텍스트를 제공하면, 수정할 필요가 거의 없는 완벽한 초안을 얻을 수 있습니다.
