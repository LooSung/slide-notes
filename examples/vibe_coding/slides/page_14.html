구현 순서 (The Flow)

1
도메인 모델
Domain Model (Entity, Value Object)
2
레포지토리 인터페이스
Repository Interface (Abstraction)
3
서비스 계층
Service Layer (Business Logic)
4
레포지토리 구현
Repository Implementation (DB)
5
컨트롤러
Controller / API Endpoint
6
의존성 주입
Dependency Injection (Wiring)

핵심 원칙

작은 단위로 분해

거대한 기능을 한 번에 구현하려 하면 길을 잃습니다. 검증 가능한 최소 단위로 쪼개서 정복하세요.

빠른 검토 (Fail Fast)

AI가 생성한 코드는 완벽하지 않습니다. 각 단계마다 실행해보고 즉시 피드백을 주고 수정하세요.

문서 동기화

구현 중에 설계가 바뀌었나요? 코드를 수정하고 문서도 바로 업데이트하세요. 나중은 없습니다.
